GIT CHEAT SHEET
[>=========<>========<>========<>========<]

Quick daily flow:
1) git status
2) git pull --rebase
3) make changes
4) git add -p
5) git commit -m "message"
6) git push


1. SETUP
git config --global user.name "Your Name"         -> set author name
git config --global user.email "you@example.com"  -> set author email
git config --global init.defaultBranch main       -> default branch name
git config --global core.editor "nano"            -> default editor
git config --global pull.rebase true              -> pull with rebase by default
git config --global rebase.autoStash true         -> auto-stash on rebase/pull


2. START A REPO
git init                     -> create repository in current folder
git clone <url>              -> clone remote repository
git status                   -> show working tree + staging state
git remote -v                -> list remotes
git remote add origin <url>  -> add remote named origin


3. STAGING AND COMMITS
git add <file>               -> stage one file
git add .                    -> stage all tracked/untracked changes
git add -p                   -> stage changes hunk by hunk
git restore --staged <file>  -> unstage file
git commit -m "message"      -> commit staged changes
git commit --amend           -> edit last commit (message/content)


4. SEE WHAT CHANGED
git diff                     -> unstaged changes
git diff --staged            -> staged changes
git diff HEAD~1..HEAD        -> changes introduced by last commit
git diff <branch1>...<branch2> -> compare branch tips from merge-base
git log                      -> full history
git log --oneline --graph --decorate --all -> compact visual history
git log --stat               -> history with changed-file stats
git log -p                   -> history with patch output
git shortlog -sn             -> commit count per author
git show <commit>            -> show one commit details


5. BRANCHES
git branch                   -> list local branches
git branch -a                -> list local + remote branches
git switch <branch>          -> switch branch
git switch -c <new-branch>   -> create and switch branch
git branch -d <branch>       -> delete merged branch
git branch -D <branch>       -> force delete local branch


6. SYNC WITH REMOTE
git fetch                    -> download remote refs (no merge)
git fetch --prune            -> remove stale remote-tracking refs
git pull                     -> fetch + merge/rebase (depends config)
git pull --rebase            -> fetch + rebase local commits
git push                     -> push current branch
git push -u origin <branch>  -> push and set upstream
git push origin --delete <branch> -> delete remote branch


7. MERGE AND REBASE
# Merge keeps branch structure
git merge <branch>           -> merge <branch> into current branch
git merge --abort            -> cancel in-progress merge

# Rebase rewrites local commit base (clean history)
git rebase main              -> replay current branch on top of main
git rebase -i HEAD~3         -> interactive rebase last 3 commits
git rebase --continue        -> continue after resolving conflicts
git rebase --abort           -> cancel in-progress rebase

Rule: avoid rebasing commits already shared on team branches.


8. CONFLICT RESOLUTION
git status                   -> see conflicted files
# edit files and resolve markers: <<<<<<< ======= >>>>>>>
git add <file>               -> mark file resolved
git diff --name-only --diff-filter=U -> list only conflicted files
git merge --continue         -> continue merge (if needed)
git rebase --continue        -> continue rebase (if rebasing)

Use ours/theirs quickly:
git checkout --ours <file>   -> keep current branch version
git checkout --theirs <file> -> keep incoming version


9. UNDO (SAFE FIRST)
git restore <file>           -> discard unstaged file changes
git restore .                -> discard all unstaged working-tree changes
git restore --source=HEAD~1 <file> -> restore file from older commit
git revert <commit>          -> create inverse commit (safe on shared history)
git revert --no-commit <commit> -> revert but keep changes uncommitted

git reset --soft HEAD~1      -> undo last commit, keep staged
git reset --mixed HEAD~1     -> undo last commit, keep unstaged (default)
git reset --hard HEAD~1      -> undo last commit + discard changes (danger)
git clean -fd                -> remove untracked files/dirs (danger)


10. STASH
git stash                    -> stash tracked changes
git stash -u                 -> stash including untracked files
git stash push -m "msg"      -> named stash
git stash list               -> list stashes
git stash show -p stash@{0}  -> preview stash diff
git stash apply stash@{0}    -> apply stash, keep it
git stash pop                -> apply latest stash and drop it
git stash drop stash@{0}     -> delete one stash
git stash clear              -> delete all stashes


11. CHERRY-PICK
git cherry-pick <commit>     -> apply one commit onto current branch
git cherry-pick A B C        -> apply multiple specific commits
git cherry-pick --no-commit <commit> -> apply changes without committing
git cherry-pick --continue   -> continue after conflict resolution
git cherry-pick --abort      -> cancel cherry-pick


12. TAGS
git tag                      -> list tags
git tag v1.0.0               -> lightweight tag
git tag -a v1.0.0 -m "v1.0.0" -> annotated tag
git tag --sort=-creatordate  -> list newest tags first
git push origin v1.0.0       -> push one tag
git push origin --tags       -> push all tags
git tag -d v1.0.0            -> delete local tag
git push origin --delete v1.0.0 -> delete remote tag


13. DEBUGGING AND RECOVERY
git blame <file>             -> line-by-line author info
git blame -L 20,40 <file>    -> blame specific line range
git grep "pattern"           -> search tracked files

git bisect start             -> begin binary search for bad commit
git bisect bad               -> mark current commit bad
git bisect good <commit>     -> mark known-good commit
git bisect reset             -> end bisect session

git reflog                   -> history of HEAD movements
git fsck --full              -> check repository integrity
# recovery example:
# 1) git reflog
# 2) find old commit hash
# 3) git reset --hard <hash>   (only if you want full rollback)


14. SUBMODULES (IF USED)
git submodule add <url> <path> -> add submodule
git submodule update --init --recursive -> fetch/init submodules
git submodule update --remote  -> move submodule to latest tracked remote commit
git clone --recurse-submodules <url> -> clone repo with submodules


15. FEATURE BRANCH WORKFLOW
# create feature branch
git switch main
git pull --rebase
git switch -c feature/my-change

# work and commit
git add -p
git commit -m "Implement my change"
git push -u origin feature/my-change

# before merge
git switch feature/my-change
git fetch origin
git rebase origin/main

# merge (example local merge flow)
git switch main
git pull --rebase
git merge --no-ff feature/my-change
git push

# cleanup
git branch -d feature/my-change
git push origin --delete feature/my-change


16. WORKTREES (MULTIPLE CHECKOUTS)
git worktree list            -> list worktrees
git worktree add ../repo-fix -b fix/bug-123 -> new worktree + branch
git worktree remove ../repo-fix -> remove worktree


17. COMMON ALIASES (OPTIONAL)
git config --global alias.st status
git config --global alias.co switch
git config --global alias.br branch
git config --global alias.cm "commit -m"
git config --global alias.lg "log --oneline --graph --decorate --all"

18. IMPORTANT TIPS
- Pull with `--rebase` on feature branches to keep history linear.
- Use `git add -p` to avoid accidental commits.
- Prefer `git revert` for shared branches; prefer `reset` for local-only history fixes.
- Before destructive cleanup, preview first:
  `git clean -nd` (dry run), then `git clean -fd` if correct.
- Before risky operations, create a safety branch:
  `git branch backup/<date>-before-rewrite`.
- If you lose work, check `git reflog` first; recovery is often possible.
- Keep commits small and single-purpose for easier review and rollback.
- Never force-push shared branches unless the team explicitly agreed.

19. PATCHES, FORKS, PICKS, SQUASHES (IMPORTANT)
Patches:
git diff > change.patch                -> create patch from unstaged changes
git diff --staged > staged.patch       -> create patch from staged changes
git format-patch -1 HEAD               -> patch file from last commit
git apply change.patch                 -> apply patch (no commit metadata)
git apply --check change.patch         -> dry-run patch apply
git am 0001-some-change.patch          -> apply commit patch with author/message

Fork workflow tips:
- Add your fork as `origin`, upstream project as `upstream`.
git remote add upstream <upstream-url> -> track original repository
git fetch upstream                     -> fetch new upstream changes
git rebase upstream/main               -> rebase feature branch on latest upstream
git push -u origin <branch>            -> push branch to your fork
- Open PR from `fork:<branch>` to `upstream:main`.

Cherry-pick tips:
- Cherry-pick only self-contained commits.
- Prefer picking small commits to reduce conflicts.
git cherry-pick -x <commit>            -> records source commit hash in message
git cherry-pick --no-commit <commit>   -> inspect/edit before committing

Squash tips:
git rebase -i HEAD~N                   -> squash last N commits interactively
git merge --squash <branch>            -> squash-merge a branch without merge commit
- Squash before PR merge when history is noisy.
- Do not rewrite/squash commits others are already using unless coordinated.

20. PANIC SITUATIONS (FAST RECOVERY)
Committed to wrong branch:
git switch correct-branch
git cherry-pick <wrong-commit>         -> move commit to right branch
git switch wrong-branch
git reset --hard HEAD~1                -> remove mistaken commit locally

Accidentally ran hard reset / lost commit:
git reflog                             -> find previous HEAD
git reset --hard <hash>                -> restore state

Need to discard all local edits quickly:
git restore .
git clean -fd                          -> remove untracked files/dirs

Pulled bad merge/rebase and want out:
git merge --abort                      -> cancel merge in progress
git rebase --abort                     -> cancel rebase in progress

21. DAILY LIFE TIPS
- Run `git status` before and after each commit/pull/push.
- Prefer `git add -p` and small commits; debugging becomes easier.
- Use branch names by intent: `fix/...`, `feat/...`, `chore/...`.
- Rebase feature branches on `main` frequently to reduce conflict size.
- Before force push, run `git log --oneline --graph --decorate -20`.

Tip: Run `git help <command>` for full docs, e.g. `git help rebase`.
